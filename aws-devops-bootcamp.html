3-Months Weekend Bootcamp Plan - Starting 22nd July 2023

Month 1:

Week 1 (Saturday):

  • Introduction to Git: Basics of version control, creating repositories, branching, merging, and resolving conflicts.
  • Hands-on project 1: Set up a Git repository, create branches, make changes, merge them, and resolve conflicts.
  • Hands-on project 2: Collaborative Git workflow: Work in teams to simulate a collaborative development environment, utilizing branching, merging, and pull requests.
  • Best Practices Session: Best practices for using Git, including commit messages, branch naming conventions, and code review techniques.
  • Interview Preparation Tips: Common Git-related interview questions and tips for effectively showcasing your Git knowledge.

Week 2 (Sunday):

  • Introduction to DevOps: Understanding the principles, benefits, and practices of DevOps.
  • Hands-on project 1: Set up a simple CI/CD pipeline using Git, perform automated testing, and deploy a basic application.
  • Hands-on project 2: Continuous Deployment: Expand the CI/CD pipeline to automatically deploy the application to production upon successful testing.
  • Best Practices Session: Best practices for building effective CI/CD pipelines, including code quality checks, automated testing strategies, and deployment strategies.
  • Interview Preparation Tips: Common DevOps-related interview questions and tips for showcasing your understanding of DevOps principles and practices.

Week 3 (Wednesday):

Doubts Clarification and Student Demos: A dedicated session to address any doubts or questions from the students regarding the topics covered in the week. Students can also showcase their projects or demos related to Git and DevOps.

Week 3 (Saturday):

  • Introduction to DevSecOps: Integrating security practices into the DevOps workflow.
  • Hands-on project 1: Enhance the CI/CD pipeline from the previous week with security checks, vulnerability scanning, and code analysis.
  • Hands-on project 2: Infrastructure Security Testing: Perform security testing and vulnerability scanning on the deployed infrastructure and application.
  • Best Practices Session: Best practices for implementing DevSecOps, including secure coding practices, security testing tools, and secure deployment strategies.
  • Interview Preparation Tips: Common DevSecOps-related interview questions and tips for demonstrating your knowledge of secure software development and deployment.

Week 4 (Sunday):

  • Introduction to AWS Cloud key services: Overview of key AWS services like EC2, S3, RDS, and IAM.
  • Hands-on project 1: Set up an EC2 instance, configure security groups, and deploy a sample application.
  • Hands-on project 2: High Availability and Fault Tolerance: Configure a multi-region setup with load balancing and failover mechanisms for the application.
  • Best Practices Session: Best practices for using key AWS services, including security configurations, scalability considerations, and cost optimization.
  • Interview Preparation Tips: Common interview questions related to AWS services and tips for effectively showcasing your knowledge of AWS Cloud.

Month 2:

Week 5 (Saturday):

  • Serverless Computing: Introduction to serverless architecture and AWS Lambda.
  • Hands-on project 1: Build and deploy a serverless application using AWS Lambda, API Gateway, and DynamoDB.
  • Hands-on project 2: Event-Driven Architecture: Implement event-driven workflows using AWS Lambda, SNS, and SQS.
  • Best Practices Session: Best practices for serverless architecture, including function design, security considerations, and performance optimization.
  • Interview Preparation Tips: Common interview questions related to serverless computing and tips for effectively showcasing your knowledge of AWS Lambda and related services.

Week 6 (Sunday):

  • Infrastructure as Code with Terraform: Introduction to Infrastructure as Code and Terraform.
  • Hands-on project 1: Use Terraform to provision and manage AWS resources, such as EC2 instances, S3 buckets, and IAM roles. Implement load balancing and auto-scaling.
  • Hands-on project 2: Infrastructure Modularity: Implement reusable modules in Terraform for different infrastructure components and deploy them across multiple environments.
  • Best Practices Session: Best practices for infrastructure as code, including modularization, version control, and state management.
  • Interview Preparation Tips: Common Terraform-related interview questions and tips for demonstrating your proficiency in using Terraform for infrastructure provisioning.

Week 7 (Wednesday):

Doubts Clarification and Student Demos: A dedicated session to address any doubts or questions from the students regarding the topics covered in the week. Students can also showcase their projects or demos related to serverless computing and infrastructure as code with Terraform.

Week 7 (Saturday):

  • Automation with Ansible: Introduction to Ansible for configuration management and automation.
  • Hands-on project 1: Create Ansible playbooks to automate the deployment and configuration of a multi-tier application. Implement blue-green deployment strategy.
  • Hands-on project 2: Configuration Drift Remediation: Implement Ansible automation to detect and remediate configuration drift across multiple servers in a fleet.
  • Best Practices Session: Best practices for Ansible automation, including playbook organization, idempotency, and error handling.
  • Interview Preparation Tips: Common Ansible-related interview questions and tips for showcasing your expertise in Ansible configuration management and automation.

Month 3:

Week 9 (Saturday):

  • Containerization with Docker: Understanding Docker, containerization, and Dockerfile creation.
  • Hands-on project 1: Containerize an application using Docker, build custom images, and deploy containers. Implement container orchestration with Docker Compose.
  • Hands-on project 2: Microservices Architecture: Dockerize and orchestrate multiple microservices using Docker Swarm.
  • Best Practices Session: Best practices for Docker containerization, including image optimization, security considerations, and container networking.
  • Interview Preparation Tips: Common interview questions related to Docker and tips for effectively showcasing your knowledge of containerization.

Week 10 (Sunday):

  • Kubernetes: Introduction to Kubernetes and container orchestration.
  • Hands-on project 1: Set up a Kubernetes cluster, deploy containers, and manage the cluster using kubectl.
  • Hands-on project 2: Scaling and Self-Healing: Implement scaling and self-healing mechanisms in a Kubernetes cluster.
  • Best Practices Session: Best practices for Kubernetes deployment, including pod design, service discovery, and cluster security.
  • Interview Preparation Tips: Common Kubernetes-related interview questions and tips for demonstrating your knowledge of container orchestration with Kubernetes.

Week 11 (Saturday):

  • Monitoring and Logging: Introduction to monitoring and logging in cloud-based environments.
  • Hands-on project 1: Set up monitoring and logging solutions using AWS CloudWatch, Prometheus, and Grafana.
  • Hands-on project 2: Log Analysis and Alerting: Implement log analysis and alerting using ELK Stack (Elasticsearch, Logstash, Kibana) or similar tools.
  • Best Practices Session: Best practices for monitoring and logging, including log aggregation, metric collection, and alerting strategies.
  • Interview Preparation Tips: Common interview questions related to monitoring and logging in cloud environments and tips for showcasing your understanding of monitoring and log analysis.

Week 12 (Sunday):

  • Cloud Application and Infrastructure Security: Best practices and techniques for securing cloud-based applications and infrastructure.
  • Hands-on project 1: Implement security measures like encryption, access control, and intrusion detection for the deployed application. Perform security auditing and monitoring.
  • Hands-on project 2: Secure Containerization: Harden container images, implement image scanning, and enforce security policies in a containerized environment.
  • Best Practices Session: Best practices for cloud application and infrastructure security, including secure network configurations, identity and access management, and encryption.
  • Interview Preparation Tips: Common interview questions related to cloud security and tips for effectively showcasing your understanding of cloud security best practices.

Week 13 (Wednesday):

Doubts Clarification and Student Demos: A dedicated session to address any doubts or questions from the students regarding the topics covered in the week. Students can also showcase their projects or demos relevant to the topics covered in the bootcamp.

Week 13 (Saturday):

  • Recap and Final Project: Review and consolidate knowledge gained throughout the bootcamp, and work on a final project that combines various technologies covered. Build a complex application or infrastructure solution that incorporates multiple concepts and best practices learned during the bootcamp.
